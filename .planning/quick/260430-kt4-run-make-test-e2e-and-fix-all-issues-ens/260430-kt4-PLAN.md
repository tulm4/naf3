# PLAN: Fix `make test-e2e` — env vars as Makefile arguments

## Problem

`make test-e2e` passes `E2E_DOCKER_MANAGED=1`, `BIZ_BINARY=...`, etc. to `$(GOTEST)` **as positional arguments** instead of **as environment variables**. The `go test` command interprets `E2E_DOCKER_MANAGED=1` as a malformed package path, producing:

```
malformed import path "E2E_DOCKER_MANAGED=1": invalid char '='
```

The docker compose infrastructure started correctly, but `SharedHarness()` returned `nil` (compose could not start, but the Makefile tear-down command still ran).

## Root Cause

```make
test-e2e: build
    docker compose ...
    @$(GOTEST) -v \
        E2E_DOCKER_MANAGED=1 \   # ← passed as argument, not env var
        BIZ_BINARY=$(BIZ_BINARY) \

$(GOTEST) = $(GOCMD) test  # go test E2E_DOCKER_MANAGED=1 ./test/e2e/...
```

The `\` line continuation puts `E2E_DOCKER_MANAGED=1` after `go test -v` as a positional arg.

## Fix

Prefix all env var assignments with `E2E_DOCKER_MANAGED=1` etc. with `E2E_DOCKER_MANAGED=1 $(GOTEST) ...` and pass the test pattern as the final positional argument. Also add `--testify-m` to show test names in verbose output.

```make
	@echo "$(YELLOW)Starting docker compose infrastructure...$(NC)"

		./test/e2e/... \
	|| { docker compose -f compose/dev.yaml down --remove-orphans; exit 1; }
	@echo "$(YELLOW)Tearing down docker compose infrastructure...$(NC)"
	docker compose -f compose/dev.yaml down --remove-orphans
	@echo "$(GREEN)E2E tests complete$(NC)"
```

The env vars `E2E_DOCKER_MANAGED=1`, `BIZ_BINARY=...`, `HTTPGW_BINARY=...`, `AAAGW_BINARY=...` must precede `$(GOTEST)` in the command. The test pattern `./test/e2e/...` is the final positional argument to `go test`.
